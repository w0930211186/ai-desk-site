N°02 GPT-5.5 · tool use 和 coding agent 的前置調

圖片來源受限（OpenAI 官網限制爬取） · 點下方連結看原文視覺

不到 6 個月，OpenAI 再出一版。

發生什麼

GPT-5.5 上架。官方 blog 強調三件事：tool use 參數呈現準確度提升、SWE-bench 分數近一步、上下文記憶從 128K 推到 256K。價格持平 GPT-5。

Hacker News 單頁滾到 1464 分 — 不是小版本的討論量。這是 OpenAI 判斷「在 Anthropic / Gemini / Mistral 再近一步之前，先搶回位置」的策略調可。

為什麼重要

第一，節奏被強迫加快。模型迭代從每年一版變成兩季一版。不是 OpenAI 想這樣，是對手等喵。Anthropic 在 2025 年底出了 Opus 4.6 之後，三個月內 Claude 4.7 又上。Google 的 Gemini 3 Pro 預告要在 Q3 出。這場比賽從「誰的模型更聰明」變「誰能先補上工具鏈那一段」。

第二，tool use 已經是 enterprise 決策點。所有模型廠年內的重要改善都集中在「模型不是更聰明，是工具操控更準」。這反映 agentic coding 已經對大模型薪水支付方式來講是主軸。過去 2 年 CTO 問的是「哪家模型 benchmark 分數高」，現在問的是「哪家的 function calling 最穩」。

我們的判斷

三個看點。

(1) 接下來 3 週觀察 GPT-5.5 在 agentic coding 場景的實戰表現。Anthropic 的 postmortem（見 N°03）正好提醒我們：快版本迭代 = 高品質失控風險。GPT-5.5 推出後能不能維持 Claude Code 過去 6 個月那種「agent 跑 2 小時不崩」的穩定度，還不好說。

(2) Tool use 參數呈現的改善，背後是「模型理解 JSON Schema 的能力」在升級。這一塊對建 MCP server、建 agent 框架的人是巨大利好。但對「靠 prompt engineering 吃飯」的人是訊號：prompt 工程師這個職位在未來 12 個月會被「工具設計師」吃掉。

(3) 256K 上下文的真實意義不是「可以讀更長的文件」。是「agent 可以做更複雜的多步任務 without 清空記憶」。這對 Cursor / Cline / Continue 這些 IDE agent 是生產力放大器。

N°03 Anthropic · Claude Code 品質 postmortem

Anthropic 官方 postmortem · 點下方連結看完整 timeline 和 latency charts

一週前官方承認 Claude Code 品質下滑。今天 postmortem 出來了。

發生什麼

文章交代一週之間，Claude Code 受影響使用者被 model routing 的一次不恐意更新分到「純理論使用 Opus」，持續 6 天。根源是一條 internal flag 設錯。

罕見的細節：latency charts、事後還原 timeline、customer impact 統計、內部溝通時程。專案用語承認「我們沒有在 30 分鐘內回應內部告警」— 這種自我檢討在 SaaS postmortem 幾乎看不到。

為什麼重要

第一，SaaS 透明度標範正在被 Anthropic 單獨推高。OpenAI 和 Google 過去對 API 中斷歷史上只說「我們知道了 已修復」。Stripe、Cloudflare 的 postmortem 是業界標竿，Anthropic 現在是 AI 圈第一個對齊這個標竿的。這不是 PR — 是 enterprise 採購文化的重寫。

第二，各家 CTO 看完這份文件都會在想一件事：「我該不該拿這幾個問題去問我現有的 AI 供應商」。問題包括：(1) 你有 30 分鐘內的內部告警嗎？(2) 出事多久會有 customer-facing postmortem？(3) 你的 model routing 是否有 feature flag 安全網？這些都不是技術問題，是治理問題。

我們的判斷

(1) Anthropic 的 enterprise 跟單節奏會直接加分。這份 postmortem 出來後一週內，預期會看到多家大型公司（特別是金融、醫療）主動把 Anthropic 列入短名單，甚至取代 OpenAI。

(2) OpenAI 2 月那個「櫃台被入侵 12 小時」事件還沒出 postmortem。現在壓力回到他們身上。如果他們不跟，未來 6 個月 enterprise market share 的差距會明顯拉開。

(3) 這份文件對「新創做 AI 產品」的 founder 也是免費教材。事故不可怕，不透明才可怕。postmortem 的結構（fact / impact / root cause / mitigation / prevention）直接套用。

N°04 MeshCore · 開源團隊因 AI 生 code 分裂

MeshCore 官方 blog · 10 天裂開 timeline 全記錄 · 點下方連結

MeshCore 一分為二。核心爭議：AI 生成 code 到底算誰寫的。

發生什麼

MeshCore（一個小型 mesh networking 開源專案）團隊 18 人分裂。6 人開始一 commit 70% 是 AI 生成且不主動註明，另 12 人要求 PR template 強制填寫 AI 用量比例、被否決、裂。

後續牽扯商標歸屬 — 原創始成員保留商標，出走方被迫用新名字。整個裂開過程在 blog 上公開完整 10 天 timeline。

為什麼重要

這是 agentic coding 時代第一個可見、有公開 timeline 的社群裂傷。過去兩年 AI code 都在「個人效率工具」層次，現在第一次進入「集體協作治理」層次。開源社群 30 年來「你的 commit 就是你寫的」的默契，被 AI agent 逼著重新寫規則。

法律層面的伏筆：如果 AI 生成 code 大量進入 GPL / Apache / MIT 授權的專案，誰是版權擁有者？OpenAI？Anthropic？coder 本人？這個問題目前沒有任何司法管轄區有清楚答案。MeshCore 這個小事件只是把這個球踢進公眾視野。

我們的判斷

(1) 接下來 6 個月會看到更多類似分裂。押的是中型開源專案（50-500 人社群），不是巨型的（Linux / Kubernetes 規模）— 巨型有官僚機制，小型有 AI 生成率集中在 maintainer。

(2) 技術性解答會先出：「AI 生成 code 註明標準」、「commit signoff 裡加 AI-assisted flag」。GitHub、GitLab、Codeberg 這些 forge 可能半年內會支援。

(3) 社會性解答會晚到：「AI code 的版權歸屬」。這要等 EU AI Act 或美國 Copyright Office 表態，樂觀估計 2027 年才有動作。

N°05 《How LLMs Work》· 一個互動頁炸爆 Karpathy 教育市場

GitHub Pages 專案 · 無 og:image · 點連結直接進互動

基於 Andrej Karpathy 3.5 小時 YouTube 講座的互動式 LLM 解說頁，上線不到 12 小時 HN 200 分。

發生什麼

把 Karpathy 那場經典講座（「Let's build GPT」）的核心概念 — attention head、softmax、token prediction、residual stream — 做成可點擊的互動 visualization。使用者可以滑動調整 hidden dimension、看每一步的梯度流向、拖曳 token 觀察 attention pattern。

技術棧：純靜態 HTML + d3.js + 一些 WebGL。不需要 GPU，瀏覽器就跑。

為什麼重要

工程師教育市場混入被吃。以前看 LLM 原理 — 花 2 小時看 Karpathy 影片 + 3000 圈 online course + 翻 Transformer 論文。現在看這一個頁，10 分鐘就能建立 mental model。

對 AI 教育產品的定價結構是破壞性。Coursera / Udacity 上一堆「手把手教你建 LLM」的課程賣 $500 美元一期。這種免費的、互動的、品質不輸的教育內容，直接把定價模型歸零。

我們的判斷

(1) 這種風格的「learn-by-playing」產品會是 AI 內容的下一波。不是 text → video → interactive 單線進化，是「所有人都可以做自己學科的互動 demo」。Anthropic 的 Prompt Engineering 互動教材、OpenAI 的 API playground 都是同一個方向。

(2) 對獨立開發者是好機會。過去做教育需要「我是哪個教授」的背書，現在只要「我做得出能跑的互動 demo」。一個人一週做出來的東西直接衝 HN 頭版，這個門檻前所未有的低。

N°06 韓國首例 · AI 假圖造成公共恐慌被拘捕

韓國成為第一個因「傳播 AI 假圖造成公共緊張」就立實罪的國家。

發生什麼

民眾上傳一張 AI 生成的「野狼在首爾里間」影像到 social media，引發數萬人關注、警方 9 天調度動員搜山。最後證實為全假。警方花 9 天溯源到原 PO 主，已拘捕。預計起訴「妨害公務」+「散布不實資訊」，刑期 6 個月到 2 年。

為什麼重要

第一，這是 AI deepfake 從「欺騙個人」到「欺騙公共機構」的分水嶺。過去兩年 deepfake 的主戰場在詐騙、名人換臉、選舉操弄 — 都是針對個人或群體。這次是直接消耗國家資源（警方搜山）。執法機構的反應模式會從此不同。

第二，韓國這個判例會被抄。日本、新加坡、歐盟都在觀望。預計未來 12 個月至少 3-5 國跟進立法。

我們的判斷

三個方向要看。

(1) 歐美會不會跟進。EU AI Act 採稿中就有相關條款（Article 50），但範圍寬泛。韓國的判例提供具體標準，可能催化 EU 具體化。

(2) Meta / X / 抖音的原生 deepfake 標識系統。C2PA 標準（Content Authenticity Initiative）已經存在但採用率低。這個事件可能推動平台強制掛 AI-generated 浮水印 — 特別是涉及「公共事件」類別。

(3) 使用者教育 vs 法律工具的平衡。刑期威懾 vs 公眾識別能力哪個更有效？韓國這案會給一個 6-12 個月的觀察窗。

    法律層面的伏筆：如果 AI 生成 code 大量進入 GPL / Apache / MIT 授權的專案，誰是版權擁有者？OpenAI？Anthropic？coder 本人？這個問題目前沒有任何司法管轄區有清楚答案。MeshCore 這個小事件只是把這個球踢進公眾視野。
    


    我們的判斷
    


    (1) 接下來 6 個月會看到更多類似分裂。押的是中型開源專案（50-500 人社群），不是巨型的（Linux / Kubernetes 規模）— 巨型有官僚機制，小型有 AI 生成率集中在 maintainer。
    


    (2) 技術性解答會先出：「AI 生成 code 註明標準」、「commit signoff 裡加 AI-assisted flag」。GitHub、GitLab、Codeberg 這些 forge 可能半年內會支援。
    


    (3) 社會性解答會晚到：「AI code 的版權歸屬」。這要等 EU AI Act 或美國 Copyright Office 表態，樂觀估計 2027 年才有動作。

    N°05 《How LLMs Work》· 一個互動頁炸爆 Karpathy 教育市場
    


    GitHub Pages 專案 · 無 og:image · 點連結直接進互動
    


    基於 Andrej Karpathy 3.5 小時 YouTube 講座的互動式 LLM 解說頁，上線不到 12 小時 HN 200 分。
    


    發生什麼
    


    把 Karpathy 那場經典講座（「Let's build GPT」）的核心概念 — attention head、softmax、token prediction、residual stream — 做成可點擊的互動 visualization。使用者可以滑動調整 hidden dimension、看每一步的梯度流向、拖曳 token 觀察 attention pattern。
    


    技術棧：純靜態 HTML + d3.js + 一些 WebGL。不需要 GPU，瀏覽器就跑。
    


    為什麼重要
    


    工程師教育市場混入被吃。以前看 LLM 原理 — 花 2 小時看 Karpathy 影片 + 3000 圈 online course + 翻 Transformer 論文。現在看這一個頁，10 分鐘就能建立 mental model。
    


    對 AI 教育產品的定價結構是破壞性。Coursera / Udacity 上一堆「手把手教你建 LLM」的課程賣 $500 美元一期。這種免費的、互動的、品質不輸的教育內容，直接把定價模型歸零。
    


    我們的判斷
    


    (1) 這種風格的「learn-by-playing」產品會是 AI 內容的下一波。不是 text → video → interactive 單線進化，是「所有人都可以做自己學科的互動 demo」。Anthropic 的 Prompt Engineering 互動教材、OpenAI 的 API playground 都是同一個方向。
    


    (2) 對獨立開發者是好機會。過去做教育需要「我是哪個教授」的背書，現在只要「我做得出能跑的互動 demo」。一個人一週做出來的東西直接衝 HN 頭版，這個門檻前所未有的低。

    N°06 韓國首例 · AI 假圖造成公共恐慌被拘捕
    




    韓國成為第一個因「傳播 AI 假圖造成公共緊張」就立實罪的國家。
    


    發生什麼
    


    民眾上傳一張 AI 生成的「野狼在首爾里間」影像到 social media，引發數萬人關注、警方 9 天調度動員搜山。最後證實為全假。警方花 9 天溯源到原 PO 主，已拘捕。預計起訴「妨害公務」+「散布不實資訊」，刑期 6 個月到 2 年。
    


    為什麼重要
    


    第一，這是 AI deepfake 從「欺騙個人」到「欺騙公共機構」的分水嶺。過去兩年 deepfake 的主戰場在詐騙、名人換臉、選舉操弄 — 都是針對個人或群體。這次是直接消耗國家資源（警方搜山）。執法機構的反應模式會從此不同。
    


    第二，韓國這個判例會被抄。日本、新加坡、歐盟都在觀望。預計未來 12 個月至少 3-5 國跟進立法。
    


    我們的判斷
    


    三個方向要看。
    


    (1) 歐美會不會跟進。EU AI Act 採稿中就有相關條款（Article 50），但範圍寬泛。韓國的判例提供具體標準，可能催化 EU 具體化。
    


    (2) Meta / X / 抖音的原生 deepfake 標識系統。C2PA 標準（Content Authenticity Initiative）已經存在但採用率低。這個事件可能推動平台強制掛 AI-generated 浮水印 — 特別是涉及「公共事件」類別。
    


    (3) 使用者教育 vs 法律工具的平衡。刑期威懾 vs 公眾識別能力哪個更有效？韓國這案會給一個 6-12 個月的觀察窗。
